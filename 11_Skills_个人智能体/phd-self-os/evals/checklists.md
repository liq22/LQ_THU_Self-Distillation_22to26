# Final Checklists

## Grounding checklist

Before final output:

- [ ] 关键事实是否来自知识库或用户提供材料？
- [ ] 日期、论文标题、作者、项目名、机构名是否核对？
- [ ] 是否区分事实、解释、推断和缺失信息？
- [ ] 是否避免把没有来源的内容写成确定事实？
- [ ] 是否添加了必要的 Obsidian 来源链接或来源说明？

## Voice checklist

- [ ] 是否中文优先，必要时保留专业英文术语？
- [ ] 是否克制、具体、诚实？
- [ ] 是否少空话、少套话、少 AI 腔？
- [ ] 是否有具体场景、行动或判断标准？
- [ ] 是否避免过度拔高或自我神化？
- [ ] 是否保留必要的不确定性和边界？

## Privacy checklist

- [ ] 是否适合目标受众？
- [ ] 是否暴露了未公开论文、数据、反馈或私人关系？
- [ ] 是否匿名化第三方？
- [ ] 是否避免将私人批评公开化？
- [ ] 是否删除了不必要的路径、附件名、截图信息？

## Legacy checklist

- [ ] 后来者是否知道第一步做什么？
- [ ] 是否列出了常见坑？
- [ ] 是否给出判断标准而不只是经历叙述？
- [ ] 是否说明了适用边界？
