# Workflow: Use the Vault

## Input

- User request.
- Intended audience: private / lab / public / application / thesis committee.
- Desired output: note / essay / checklist / answer / outline / email / speech / guide.
- Source scope if specified.

## Steps

1. Determine task class.
2. Locate dashboard entry point.
3. Follow links to primary notes.
4. Extract facts, interpretations, and gaps.
5. Draft using `config/voice_profile.md`.
6. Add links or source notes.
7. Run `evals/checklists.md`.

## Output

Use this minimal structure when unsure:

```markdown
# 标题

## 一句话结论

## 依据

## 展开

## 不确定或待补充

## 相关笔记
```
