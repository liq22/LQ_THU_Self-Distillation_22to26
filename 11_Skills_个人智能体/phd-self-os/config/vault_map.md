# Vault Map

## Repository root

The knowledge base is organized as an Obsidian-compatible GitHub repository.

Core folders:

```text
00_Inbox_收集箱/
01_Dashboard_总览/
02_Research_科研工作/
03_Papers_论文与发表/
04_Thesis_毕业论文/
05_Life_生活与见闻/
06_Thinking_思考沉淀/
07_Feedback_反馈与复盘/
08_Legacy_传承资料/
09_Admin_Career_事务与去向/
10_Assets_附件/
11_Skills_个人智能体/
90_Templates_模板/
99_Archive_归档/
```

## Folder roles

### `00_Inbox_收集箱/`

Temporary capture. Use only when recent material has not been processed.

### `01_Dashboard_总览/`

The first stop for almost all tasks.

Key files:

- `Home.md`: main entry.
- `PhD_Timeline.md`: chronological spine.
- `Research_Map.md`: research areas, projects, conceptual relations.
- `Publication_Map.md`: paper pipeline and publication status.
- `People_Map.md`: collaborators, mentors, recurring feedback sources.
- `Graduation_Checklist.md`: degree requirements and final milestones.
- `整理看板.md`: active organization tasks.

### `02_Research_科研工作/`

Primary source for what the user actually did scientifically.

Use for:

- research claims;
- project origin stories;
- experiments and negative results;
- code/data/method decisions;
- group meeting records;
- research lessons.

### `03_Papers_论文与发表/`

Primary source for publication narratives.

Use for:

- paper histories;
- submission timelines;
- reviewer critiques;
- rebuttal strategies;
- published outputs;
- literature positioning.

### `04_Thesis_毕业论文/`

Primary source for dissertation writing and defense.

Use for:

- chapter plans;
- thesis argument;
- figures and tables;
- defense Q&A;
- graduation procedures.

### `05_Life_生活与见闻/`

Use for personal narrative and lived experience.

Good for:

- campus/city observations;
- conferences and travel;
- daily routines;
- emotional texture;
- health and sustainability.

### `06_Thinking_思考沉淀/`

Use for durable ideas.

Good for:

- research philosophy;
- work systems;
- career choices;
- conceptual notes;
- meta-reflection.

### `07_Feedback_反馈与复盘/`

Use for learning loops.

Good for:

- advisor feedback;
- peer feedback;
- reviewer feedback;
- annual/semester/monthly retrospectives;
- patterns across failures and corrections.

### `08_Legacy_传承资料/`

Use when writing for juniors or lab successors.

Good for:

- onboarding;
- lab survival guide;
- reusable checklists;
- FAQ;
- workflow documentation.

### `09_Admin_Career_事务与去向/`

Use for career-facing materials.

Good for:

- CV;
- personal statements;
- applications;
- awards;
- talks/posters;
- network records.

### `10_Assets_附件/`

Use only as supporting material. Do not rely on raw attachments unless the user grants access and the task requires it.

### `90_Templates_模板/`

Use for note creation and normalization.

### `99_Archive_归档/`

Use only after active folders are insufficient.
