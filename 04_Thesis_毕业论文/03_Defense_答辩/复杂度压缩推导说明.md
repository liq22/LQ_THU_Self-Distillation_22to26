# 复杂度压缩推导说明

> 来源：临时工作目录 L_26_6 整理
> 整理日期：2026-06-17
>
> **说明：** 本文档用于解释"为什么加入物理约束和稀疏结构后，NSNet 的经验复杂度项会下降"。

---

## 核心表达

**NSNet 并不是简单说"加约束更好"，而是通过两个层面的约束压缩有效假设空间：**

1. **结构层压缩**：从 M 个候选结构中最多激活 k 个，组合复杂度从大规模稠密结构变成稀疏结构项。
2. **参数层压缩**：物理参数被限制在有效紧致空间 Ω_eff 内，连续参数复杂度由 d_eff 控制。
3. 最后通过覆盖数上界推导到经验 Rademacher 复杂度上界，从而解释泛化误差界中复杂度项下降。

---

## 四步推导链

### Step 1：离散结构数目

**模块标题：** 稀疏拓扑：减少结构选择数

**图形内容：** 左侧画一个稠密网络小示意，右侧画一个稀疏 DAG 小示意，稠密网络用灰色，稀疏激活路径用紫色突出。

**公式：**
```
|Z|_0 ≤ k ≪ M
|A| ≤ Σ(j=0 to k) C(M,j) ≤ (eM/k)^k
```

**底部解释：** 结构复杂度由 M 降为 k·log M 量级

**结论：**
```
log|A| = O(k·log M)
```

---

### Step 2：固定结构后的参数覆盖

**模块标题：** 固定拓扑：只看连续参数自由度

**图形内容：** 画一个紫色边界的参数空间小球或低维流形，标注"物理参数可行域"

**公式：**
```
Ω_eff ⊂ R^(d_eff)
N(Ω_eff, |·|_2, ρ) ≤ (3C_eff/ρ)^(d_eff)
```

**说明：** 物理约束降低有效维度 d_eff

**注意：** 不要画成普通高维黑箱参数空间，要突出"受限流形 / 可行域"。

---

### Step 3：Lipschitz 映射到函数覆盖数

**模块标题：** 参数覆盖 → 函数覆盖

**图形内容：** 画一个从"参数空间小球"到"函数曲线族"的箭头，箭头标注"L_eff-Lipschitz"

**公式：**
```
d_S(F_(ϑ_1,a), F_(ϑ_2,a)) ≤ L_eff|ϑ_1 - ϑ_2|_2
```

**覆盖数传递关系：**
```
N(H_a, d_S, ε) ≤ (3C_eff·L_eff/ε)^(d_eff)
```

**底部解释：** 参数空间越小、映射越稳定，函数类覆盖数越小

---

### Step 4：并集上界与 Rademacher 复杂度

**模块标题：** 合并所有结构，得到复杂度上界

**图形内容：** 画多个小的结构子空间 H_a，用虚线框合并成整体

**公式：**
```
H_NSNet = ∪(a∈A) H_a
```

**覆盖数合并结果：**
```
log N(H_NSNet, d_S, ε) ≤ O(k·log M) + d_eff·log(3C_eff·L_eff/ε)
```

**最终结论（重点）：**
```
R̂_S(H_NSNet) ≤ O(√(k·log M/n) + √(d_eff·log(C_eff·L_eff·n)/n))
```

---

## 如何理解这个界？

### 结构项
```
√(k·log M/n)
```
**解释：** 由稀疏拓扑决定。k 越小，激活结构越少，组合复杂度越低。

### 参数项
```
√(d_eff·log(C_eff·L_eff·n)/n)
```
**解释：** 由有效参数维度决定。物理约束使参数搜索不再发生在无约束高维空间。

### 样本规模
```
n ↑ ⇒ R̂_S(H) ↓
```
**解释：** 样本越多，复杂度项被样本平均效应进一步压低。

## 逻辑约束的作用

右侧可以加一个窄小虚线注释框：

**逻辑相容性的作用：**
- 训练期：作为软约束改变优化解
- 推理期：作为规则满足度审查诊断边界
- 若写成硬可行集，可进一步收缩覆盖数

**注意：** 不要把逻辑约束强行画成最终复杂度界中的第三个显式加法项。它在本文中主要通过约束优化解和规则满足度影响可行解空间。

---

## 底部结论条

**结论：NSNet 的复杂度压缩来自两条主线——稀疏结构减少可选路径，物理参数降低有效自由度；二者共同降低经验复杂度项，为泛化倾向提供统计学习解释。**
