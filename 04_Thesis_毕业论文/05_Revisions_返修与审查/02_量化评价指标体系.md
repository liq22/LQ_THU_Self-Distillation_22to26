## 一、一票否决 / 暂停送审项

这些项目不按普通扣分处理。出现后应优先修正，再进入总分评价。

| 编号 | 红线项 | 判定标准 | 后果建议 |
|------|--------|----------|----------|
| G0-1 | 疑似学术不端 | 存在未标出处的大段文字、数据、图像、方法、观点复用；疑似伪造、篡改、代写 | 暂不建议送审/答辩/定稿，逐条核实 |
| G0-2 | 直接引文不规范 | 直接引用原文但未加引号，或未标出处 | 按 P0/P1 处理；必须补引号、出处或改写 |
| G0-3 | 成果归属不清 | 本人、导师、合作者、团队项目、小论文之间贡献边界不清 | 必须补充贡献说明和发表情况 |
| G0-4 | 核心概念或方法硬伤 | 关键定义、公式、实验设计、推理链条存在明显错误 | 不建议送审，先做学术性修正 |
| G0-5 | 数据/图表不可追踪 | 数据来源、实验条件、模型来源、图表来源不明 | 必须补来源、授权、实验条件和可复核说明 |
| G0-6 | 必要组成部分缺失 | 定稿阶段缺声明、个人简历/成果、指导教师评语、答辩决议书等必要后置材料 | 定稿前必须补齐 |
| G0-7 | 题目或摘要涉密 | 公开论文题目、摘要出现涉密信息 | 必须按保密要求处理 |
| G0-8 | 评阅意见回应流于表面 | 复审/重新送审时未实质回应专家核心异议 | 不建议仓促复审 |

**学术规范检查不能只依赖重复率或系统相似度。**学校方案明确要求对系统标出的疑似问题具体分析，检测结果只是辅助手段，且对违反学术规范现象采取"零容忍"态度，不得单纯设置检测合格率。

## 二、100 分总评分表

| 一级维度 | 权重 | 评分要点 | 低分表现 |
|----------|------|----------|----------|
| 硬格式与组成完整性 | 10 | 组成顺序、封面、摘要、目录、页码、篇眉、图表、参考文献、后置材料符合写作指南 | 题目超字数；摘要超页；目录未更新；页码错；后置材料缺失 |
| 选题与问题意识 | 8 | 题目具体，研究问题明确，主题单一，问题有学术或应用价值 | 题目笼统；问题不是问题；几块工作拼接 |
| 引言与文献综述 | 12 | 包含问题提出、背景意义、文献综述、研究方法、结构安排；综述有"述"有"评" | 文献堆砌；缺近年关键成果；缺研究缺口；不说明本文切入点 |
| 理论基础与方法正确性 | 12 | 概念准确，理论基础扎实，方法选择合理，公式和推理正确 | 基本概念错；方法与问题不匹配；公式/实验设计有硬伤 |
| 创新性、工作量与研究深度 | 16 | 创新点明确，工作量饱满，理论分析和实验验证充分，贡献可区分 | 创新性空泛；只是工程堆叠；研究浅；缺关键实验 |
| 结果、证据与验证充分性 | 10 | 数据可靠，实验条件清楚，对比基线合理，消融/泛化/误差分析充分 | 结论超出证据；图表和文字不一致；缺对比和消融 |
| 结构主线与章节衔接 | 10 | 各章围绕总问题展开，章节之间有递进关系，逻辑闭环 | 章节割裂；论题庞杂；像论文集或项目报告拼接 |
| 学术规范与引用归属 | 12 | 引文、图表、数据、模型、代码、已发表成果、合作成果均规范标注 | 漏引；只引次级来源；自引不说明；贡献边界模糊 |
| 语言、图表、公式与参考文献细节 | 6 | 语言严谨简明，图表清晰，术语统一，参考文献格式统一 | 错别字多；图表粗糙；坐标轴无单位；参考文献格式错 |
| 结论质量 | 4 | 结论是总体提炼，包含核心观点、创造性工作、局限和未来工作 | 各章小结重复；夸大"首次/领先"；不区分自己与他人成果 |
| **总分** | **100** | | |

## 三、分数与模拟评阅等级映射

该映射用于内部预审，不替代正式评阅。

| 总分 | 模拟等级 | 送审/答辩建议 | 风险解释 |
|------|----------|----------------|----------|
| 90–100 | A 倾向 | 可送审/可进入下一阶段 | 仅有少量 P2/P3 问题，无明显硬伤 |
| 80–89 | B 倾向 | 修改后送审较稳 | 存在若干 P1/P2 问题，但主线、创新和规范基本可控 |
| 70–79 | C 风险 | 不建议直接送审，应较大修改 | 专家可能认为创新性、深度、文献或写作不足 |
| 60–69 | 强 C / D 风险 | 暂不建议送审 | 可能存在结构性缺陷或多个关键维度不达标 |
| <60 | D 风险 | 不建议送审/答辩 | 存在重大格式、学术、方法或论证问题 |

**正式评阅中，C 或 D 意见原则上需要再次送审**，部分院系对小项评价也可能有额外要求；最终专家意见一般应为 A 或 B。

## 四、细化指标：硬格式合规

| 指标 | 量化规则 | 通过标准 | 风险等级 |
|------|----------|----------|----------|
| 论文题目 | 汉字符数 | ≤25 个汉字符 | 超出为 P1 |
| 中文摘要 | 字数与篇幅 | 800–1000 汉字（符），限 1 页 | 不满足为 P1 |
| 摘要内容 | 内容结构 | 问题、方法、过程、结果、结论齐全 | 缺结果/结论为 P1 |
| 摘要禁项 | 图表材料 | 不出现图片、图表、表格或插图材料 | 出现为 P1 |
| 关键词 | 数量与分隔 | ≤5 个；分号分隔 | 超出为 P2 |
| 中英文摘要 | 对应关系 | 内容对应，不互相增删核心结论 | 不对应为 P1 |
| 目录 | 编入范围 | 从中文摘要开始，至论文结束；列至二级节标题 | 错误为 P2 |
| 页码 | 编号规则 | 摘要至缩略语用罗马数字，正文起用阿拉伯数字 | 错误为 P2 |
| 图编号 | 编号规则 | 按章编号，如图 2.1 或图 2-1 | 混乱为 P2 |
| 表编号 | 编号规则 | 按章编号；表题在表上方 | 混乱为 P2 |
| 公式编号 | 编号规则 | 按章编号，正文引用一致 | 混乱为 P2 |
| 表格 | 格式 | 优先三线表，必要时辅助线 | 不规范为 P2 |
| 图中文字 | 语言 | 描述性文字、坐标轴、图例、表头以中文为主 | 不满足为 P2 |
| 参考文献标注 | 体例 | 顺序编码制或著者-出版年制二选一，全文统一 | 混用为 P1 |
| 参考文献表 | 对应关系 | 正文引用与文后文献一一对应 | 缺失为 P1 |
| 致谢 | 篇幅 | 限 1 页 | 超出为 P2 |
| 学术成果 | 列入规则 | 已刊载/出版/授权或正式录用方可列入 | 误列为 P1 |
| 后置材料 | 定稿要求 | 评语、决议书等定稿须补齐，不留"×××"或提示框 | 未补齐为 P1 |

## 五、细化指标：内容质量

### 5.1 引言评分表

| 子项 | 分值 | 满分标准 | 扣分标准 |
|------|------|----------|----------|
| 问题提出 | 2 | 清楚说明研究问题是什么 | 只谈背景，不提出问题 |
| 背景意义 | 2 | 说明理论意义和现实/工程意义 | 泛泛而谈，无针对性 |
| 文献综述 | 4 | 有研究脉络、代表文献、最新进展、局限评价 | 只罗列文献，无评价 |
| 研究方法 | 2 | 方法与问题匹配，说明为什么用该方法 | 方法名称堆砌 |
| 论文结构安排 | 1 | 各章功能清楚，体现递进关系 | 机械列章 |
| 本文贡献定位 | 3 | 明确研究缺口和本文贡献 | 创新点模糊或夸大 |
| **总分** | **14** | | |

**写作指南明确指出，引言通常包括问题提出、选题背景及意义、文献综述、研究方法和论文结构安排；文献综述要在"述"的同时有"评"。**

---

### 5.2 正文各章评分表

| 子项 | 分值 | 满分标准 | 高风险表现 |
|------|------|----------|------------|
| 本章问题 | 2 | 每章开头明确本章解决什么问题 | 章节像材料堆砌 |
| 方法或理论 | 3 | 方法、模型、理论推导清楚 | 方法只描述流程，缺原理 |
| 结果支撑 | 3 | 图表、实验、案例支撑本章结论 | 结论无证据 |
| 与总问题关系 | 2 | 本章贡献能回扣全文主线 | 与论文主题关系弱 |
| 与前后章衔接 | 2 | 章节之间递进自然 | 前后跳跃 |
| 本人贡献区分 | 3 | 清楚区分本人、他人、团队成果 | 贡献混写 |
| **总分** | **15/章** | | |

---

### 5.3 结论评分表

| 子项 | 分值 | 满分标准 | 扣分标准 |
|------|------|----------|----------|
| 总体提炼 | 2 | 提炼全文核心观点 | 各章小结简单重复 |
| 创造性工作 | 2 | 明确说明本人创造性贡献 | 贡献泛泛 |
| 学术/工程意义 | 1 | 说明成果地位、作用和意义 | 只有自我评价 |
| 局限性 | 1 | 客观说明研究不足 | 完全不谈局限 |
| 未来工作 | 1 | 提出合理后续方向 | 空泛口号 |
| 成果归属 | 1 | 区分本人、导师、他人成果 | 混同贡献 |
| 表述克制 | 1 | 避免无证据的"首次、领先、填补空白" | 夸大表述 |
| **总分** | **9** | | |

**写作指南明确要求结论不是各章小结的简单重复，应包括核心观点、创造性工作、研究局限和未来工作，并严格区分自己取得的成果与指导教师及他人的学术成果；没有充分证据时应避免"首次、领先、填补空白"等表述。**

## 六、修改优先级规则

| 优先级 | 定义 | 处理时限 | 示例 |
|--------|------|----------|------|
| P0 | 红线问题，不改不建议送审/答辩/定稿 | 立即处理 | 学术不端风险、核心方法错误、成果归属不清 |
| P1 | 高风险问题，可能导致 C/D 或复审 | 第一轮修改完成 | 创新性不清、文献缺口不成立、摘要不合规 |
| P2 | 中风险问题，影响专家阅读和说服力 | 第二轮修改完成 | 图表格式不统一、参考文献细节错误、章节衔接弱 |
| P3 | 低风险问题，属于润色和一致性优化 | 定稿前完成 | 个别措辞、标点、局部表达不够凝练 |

## 七、智能体输出质量要求

智能体每次 review 必须满足以下标准：

| 要求 | 合格标准 |
|------|----------|
| 有定位 | 每个问题必须指出章节、段落、图表、公式或参考文献位置 |
| 有证据 | 必须引用原文片段或描述可核查的问题形态 |
| 有规则 | 必须说明违反了哪类写作、格式、学术规范或送审风险原则 |
| 有动作 | 必须给出具体修改动作，不只写"建议完善" |
| 有优先级 | 每个问题标注 P0/P1/P2/P3 |
| 有量化 | 给出分项得分、总分和风险等级 |
| 有结论 | 明确是否建议送审、答辩或定稿 |
| 不臆造 | 不编造文献、数据、政策、实验或结论 |
