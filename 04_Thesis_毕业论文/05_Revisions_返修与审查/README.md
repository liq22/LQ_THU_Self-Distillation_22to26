## 使用场景

本目录适用于以下场景：

1. **送审前预审** - 在正式送审前对论文进行全面检查
2. **评阅意见返回后** - 根据专家评阅意见进行系统性返修
3. **答辩前准备** - 答辩前的最终格式和学术规范检查
4. **定稿前审查** - 学位审议前的完整性审查

## 审查流程

智能体审查遵循以下流程：

```
Step 0: 确认材料范围
    ↓
Step 1: P0 红线审查（一票否决项）
    ↓
Step 2: 硬格式审查
    ↓
Step 3: 内容质量审查
    ↓
Step 4: 学术规范审查
    ↓
Step 5: 送审风险分类
    ↓
输出格式：审查报告
```

---

## 核心原则

1. **不臆造** - 不得编造文献、数据、实验结果、政策要求
2. **具体化** - 必须说明"在哪里改、怎么改、预期效果"
3. **可追踪** - 所有修改必须可追踪、可解释、可撤销
4. **学术优先** - 学术规范和红线问题优先于文字润色
5. **主线统一** - 保持论文主线的统一叙事

## 评分体系

采用 100 分制评分体系，分数与模拟评阅等级映射如下：

| 总分 | 模拟等级 | 送审/答辩建议 |
|------|----------|----------------|
| 90–100 | A 倾向 | 可送审/可进入下一阶段 |
| 80–89 | B 倾向 | 修改后送审较稳 |
| 70–79 | C 风险 | 不建议直接送审，应较大修改 |
| 60–69 | 强 C / D 风险 | 暂不建议送审 |
| <60 | D 风险 | 不建议送审/答辩 |
