# 社会实践流程

## 摘要

社会实践原始材料已私有归档。GitHub 版本沉淀材料准备流程、报告结构和复盘经验，不公开企业评估表、导师评定表、签字页、内部项目细节和个人证明材料。

## 材料包结构

| 材料 | 用途 | GitHub 版本 |
|---|---|---|
| 技术报告 | 说明实践任务、方法、结果和收获 | 提供结构模板，不放原文 |
| 思想小结 | 说明实践认识和个人成长 | 提供写作提纲 |
| 导师评定表 | 学校/学院审核 | 只记录材料类型 |
| 企业评估表 | 实践单位评价 | 不公开 |
| 汇报 PPT | 答辩或总结汇报 | 提供页结构 |
| 新闻稿/风采录 | 奖学金或展示材料 | 提供脱敏写法 |

## 技术报告结构

1. 实践背景：单位、场景和问题都做泛化描述。
2. 任务目标：说明要解决的工程问题和约束。
3. 方法路线：描述技术方案、数据流程和评估方式。
4. 工作内容：按模块列出个人贡献。
5. 结果与价值：强调可验证产出，不披露企业敏感数据。
6. 问题与反思：记录协作、沟通、工程落地中的经验。
7. 后续建议：给后来同学的准备建议。

## 汇报 PPT 结构

- 封面：实践主题、单位类型、时间范围。
- 背景：行业/场景问题。
- 任务：实践中承担的具体工作。
- 方法：技术路线图。
- 结果：脱敏后的指标、案例或流程改进。
- 收获：工程认知、团队协作、科研启发。
- 后续：如何把实践经验转回研究主线。

## 准备顺序

- [ ] 先确认学院要求的必交材料和截止时间。
- [ ] 技术报告先成稿，再从中抽取 PPT。
- [ ] 导师评定和企业评估预留签字/盖章时间。
- [ ] 新闻稿或风采录只写可公开信息。
- [ ] GitHub 版只保留模板和复盘，不保留企业表格原件。

## 常见坑

| 坑 | 表现 | 处理方式 |
|---|---|---|
| 报告太像论文 | 只讲模型，不讲实践场景 | 加入任务来源、约束和协作过程 |
| 企业信息暴露 | 出现内部系统、数据、人员信息 | 用泛化场景和脱敏指标替代 |
| 材料顺序反了 | 先做 PPT，报告返工 | 先写报告骨架，再压缩成展示 |
| 签字时间不足 | 表格齐了但缺评价或签字 | 提前列出责任人和截止时间 |
