# 博士资格考试流程

## 摘要

博士资格考试原始材料已私有归档。GitHub 版本只沉淀李奇的准备流程、汇报结构和复盘经验，不公开题库原文、他人 PPT、个人路线图源文件或未授权参考材料。

## 准备路线

| 阶段 | 目标 | 输出物 |
|---|---|---|
| 规则确认 | 弄清楚考试形式、题型、评价重点和时间安排 | 一页考试规则摘要 |
| 方向定位 | 把博士主线放到学科问题中说明 | 研究问题地图 |
| 知识补齐 | 复习题库和基础概念，补足方向外短板 | 错题/概念清单 |
| 汇报准备 | 形成 10-20 分钟可讲清楚的研究路线 | 汇报 PPT 大纲 |
| 模拟问答 | 准备为什么做、怎么验证、边界在哪里 | Q&A 表 |
| 复盘归档 | 记录真实问题和准备不足 | 经验教训 note |

## 汇报 PPT 建议结构

- 个人研究背景：用 1-2 页说明方向、问题和已有基础。
- 学科定位：说明研究问题属于哪个领域，和设计所/机械工程的关系是什么。
- 核心问题：把问题压缩成 2-3 个可回答的问题。
- 方法路线：解释技术路线，不堆公式和模型细节。
- 阶段成果：只呈现已经稳定的工作，不放未确认结论。
- 后续计划：说明博士阶段如何继续推进。
- 风险与边界：主动说明数据、验证、工程应用和理论解释的限制。

## Q&A 准备表

| 问题类型 | 准备方式 |
|---|---|
| 为什么选这个问题 | 从学科价值、工程需求、个人主线三层回答 |
| 和已有工作的区别 | 准备 3 篇最相关工作，说明差异不是口号 |
| 方法是否可靠 | 准备验证设计、消融逻辑、失败案例 |
| 数据和场景是否充分 | 说明数据来源、泛化限制和后续补充计划 |
| 博士阶段能否完成 | 拆成已完成、正在做、计划做三类 |

## 李奇复盘要点

- 资格考试不是只展示成果，更重要的是展示“能否独立定义问题并推进研究”。
- 参考他人 PPT 的价值在结构，不在内容；不要把别人的研究叙事套到自己身上。
- 路线图应早做，哪怕粗糙，也能暴露问题定义不清、验证链条断裂和时间安排过满。
- 题库复习要服务于问答，不要停留在背概念。

## GitHub 上传边界

- 可提交：准备路线、PPT 结构、Q&A 类型、脱敏复盘。
- 不提交：题库原文、他人姓名材料、他人 PPT/PDF、个人 PPT 原件、学号、现场提问原话。
