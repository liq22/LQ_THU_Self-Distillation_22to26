# 博士毕业流程索引

## 摘要

用于快速定位博士毕业流程、图像采集、论文评阅、最终学术报告、答辩申请、申报学位和答辩后事项。此页面向内部共享，保留流程经验，不提交个人申请材料、评阅意见、签字扫描件和论文全文。

## 总流程

| 阶段 | 核心目标 | 典型材料 | GitHub 版本保留方式 |
|---|---|---|---|
| 毕业图像采集 | 完成毕业照片与学籍信息核对 | 图像采集通知、采集码说明、毕业预申请说明 | 保留公共说明和操作 checklist |
| 论文写作与送审 | 确认格式、学术规范和评阅材料 | 写作指南、评阅程序、评阅意见答复模板 | 保留公共规范、脱敏送审清单 |
| 最终学术报告 | 完成培养环节中的最终报告 | 申请表、记载表、创新成果审核表 | 保留 Markdown 表格索引 |
| 答辩申请 | 组建答辩委员会并提交答辩材料 | 答辩申请材料清单、答辩公告、决议书参考模板 | 保留流程和材料类型，不保留名单 |
| 申报学位 | 汇总上会和学位审批材料 | 成绩单、学位申请书、评阅材料、情况简介 | 只保留脱敏材料目录 |
| 答辩后事项 | 完成论文提交、授权、系统确认和归档 | 答辩后事项清单、授权说明 | 保留流程 checklist |

## 送审前 Checklist

- [ ] 确认论文题目、摘要、关键词、中英文信息一致。
- [ ] 按最新版写作指南检查封面、声明页、授权页、目录、图表编号和参考文献格式。
- [ ] 完成学术规范检查，保留系统结果和必要说明。
- [ ] 准备评阅意见答复模板，但不要在 GitHub 版记录原始评阅意见。
- [ ] 与导师确认送审版本、修改说明和是否有涉密/延期公开需求。

## 答辩申请 Checklist

- [ ] 确认答辩时间、地点、线上/线下形式和秘书分工。
- [ ] 准备答辩公告、答辩委员会相关材料、决议书参考模板。
- [ ] 检查评阅意见是否已全部返回并满足答辩申请条件。
- [ ] 预先准备答辩记录、出席名单、表决相关材料。
- [ ] GitHub 版只记录流程，不记录委员名单、联系方式、签字页。

## 申报学位 Checklist

- [ ] 按学院材料清单整理成绩单、学位申请书、答辩申请书、导师评语、评阅材料、上会情况简介。
- [ ] 将多个系统导出的材料按学院要求合并、命名、排序。
- [ ] 单独维护一个私有 checklist，记录每份材料的来源、版本、签字状态和提交状态。
- [ ] GitHub 版只保留材料类型和准备顺序，不保存扫描件或个人 PDF。

## 答辩后 Checklist

- [ ] 根据答辩意见修改论文并确认最终版。
- [ ] 完成学位论文使用授权、论文提交、系统状态确认。
- [ ] 检查学院/研究生院是否还需要纸质版、电子版或系统上传。
- [ ] 将最终经验整理为可复用清单，删除或私有保存签字扫描件。

## 常见坑

| 坑 | 表现 | 处理方式 |
|---|---|---|
| 材料版本混乱 | 同一材料有电子版、扫描版、待签字版 | 用阶段前缀和状态后缀命名，保留唯一提交版 |
| 清单来自多个系统 | 学院清单、研究生院清单、秘书系统输出不完全一致 | 以最终提交口径为准，另建差异记录 |
| 签字材料难回溯 | 扫描件和原件分散 | 私有区按“待签字/已签字/已提交”分层 |
| GitHub 误提交 | 评阅意见、成绩单、申请书被当作普通附件 | 只提交脱敏 Markdown，原件全部留本地私有区 |
