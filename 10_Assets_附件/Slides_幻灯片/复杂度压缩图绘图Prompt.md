## 绘图 Prompt

请生成一张 **4:3 横版博士答辩 PPT 内容页**，主题为：

**约束假设空间的复杂度压缩推导**

---

## 整体风格要求

### 配色风格
- 主色使用深清华紫，接近 #671372 / #680874，用于标题栏、章节标题、重点模块、箭头和强调线条
- 辅助色使用偏亮的紫色或玫紫色，如 #6F329D / #7233A1 / #951B7C，用于渐变背景、模块标题条和视觉过渡
- 浅紫色如 #E2C8EA / #DCC4E8 用作信息容器底色，使内容有层次但不过度抢眼
- 背景以白色或极浅灰为主，保持学术页面的干净感
- 红色仅用于"问题、误差、错误路径"等负面或警示信息，不作为主视觉色

### 版式风格
- 页面比例为 4:3，整体采用清晰的网格化排版
- 封面或章节页可使用横向大面积紫色标题带，形成稳定的视觉中心
- 正文页则采用"白底 + 紫色标题条 + 浅紫容器 + 虚线框/箭头"的结构
- 页面元素以矩形、圆角矩形、虚线容器、细线分割和单向箭头为主，强调逻辑关系、结构分组和信息流动

### 图表元素风格
- 图中模块应具有"容器感"：用紫色实线或虚线框表示逻辑边界，用浅紫半透明底色表示中间层或总结层
- 重点结论可放在紫色标题条或浅紫圆角框中
- 次要说明放在白色卡片或浅灰卡片中
- 允许使用轻微阴影，但阴影应克制，只用于增强层次，不做强烈立体效果

### 字体与文字风格
- 字体统一使用微软雅黑
- 标题使用加粗白字或深紫加粗字，正文使用深灰或黑色，重点词可用深紫加粗
- 建议标题字号明显大于正文，正文保持简洁，避免长段落
- 页面文字应呈现"答辩概括语"风格：短句、并列结构、关键词突出，不写成论文段落

### 整体视觉气质
- 风格应体现"清华博士答辩、理工科论文、神经符号网络、可解释智能诊断"的正式感
- 画面应干净、克制、层次明确，既有清华紫的学校识别度，又避免商业海报式的炫光、过度渐变和复杂装饰
- 核心目标是让观众快速看懂：问题是什么、方法结构是什么、信息如何流动、创新点在哪里

**不需要目录和标题页面。**

## 页面版式

### 顶部区域：问题引入
顶部用紫色标题条写：
**约束假设空间如何压缩复杂度？**

标题下方放一句简短说明：
**从泛化界出发，将 NSNet 的复杂度拆解为"稀疏结构项 + 有效参数项"。**

旁边可放一个小型公式卡片：
```
R(h) ≤ R̂_S(h) + 2R̂_S(H) + √(ln(1/δ)/(2n))
```

在公式旁边用箭头或标注突出：
**关键问题：如何降低 R̂_S(H)？**

---

## 中间区域：四步推导链

中间采用横向流程图，四个圆角矩形模块从左到右排列，用深紫箭头连接。每个模块上方用编号，体现推导步骤。

### Step 1：离散结构数目

**模块标题：** ① 稀疏拓扑：减少结构选择数

**图形内容：**
左侧画一个稠密网络小示意，右侧画一个稀疏 DAG 小示意，稠密网络用灰色，稀疏激活路径用紫色突出。

**公式写：**
```
|Z|_0 ≤ k ≪ M
|A| ≤ Σ(j=0 to k) C(M,j) ≤ (eM/k)^k
```

**底部解释短句：** 结构复杂度由 M 降为 k·log M 量级

**并用紫色强调：**
```
log|A| = O(k·log M)
```

---

### Step 2：固定结构后的参数覆盖

**模块标题：** ② 固定拓扑：只看连续参数自由度

**图形内容：**
画一个紫色边界的参数空间小球或低维流形，标注：
**物理参数可行域**
```
Ω_eff ⊂ R^(d_eff)
```

**公式写：**
```
N(Ω_eff, |·|_2, ρ) ≤ (3C_eff/ρ)^(d_eff)
```

**旁边用浅紫小标签说明：** 物理约束降低有效维度 d_eff

**注意：** 不要画成普通高维黑箱参数空间，要突出"受限流形 / 可行域"。

---

### Step 3：Lipschitz 映射到函数覆盖数

**模块标题：** ③ 参数覆盖 → 函数覆盖

**图形内容：**
画一个从"参数空间小球"到"函数曲线族"的箭头，箭头标注：
**L_eff-Lipschitz**

**公式写：**
```
d_S(F_(ϑ_1,a), F_(ϑ_2,a)) ≤ L_eff|ϑ_1 - ϑ_2|_2
```

然后给出覆盖数传递关系：
```
N(H_a, d_S, ε) ≤ (3C_eff·L_eff/ε)^(d_eff)
```

**底部解释短句：** 参数空间越小、映射越稳定，函数类覆盖数越小

---

### Step 4：并集上界与 Rademacher 复杂度

**模块标题：** ④ 合并所有结构，得到复杂度上界

**图形内容：**
画多个小的结构子空间 H_a，用虚线框合并成整体：
```
H_NSNet = ∪(a∈A) H_a
```

然后写覆盖数合并结果：
```
log N(H_NSNet, d_S, ε) ≤ O(k·log M) + d_eff·log(3C_eff·L_eff/ε)
```

**最后在模块底部给出最终结论公式，用深紫色重点框突出：**
```
R̂_S(H_NSNet) ≤ O(√(k·log M/n) + √(d_eff·log(C_eff·L_eff·n)/n))
```

---

## 下方区域：公式含义解释

下方放一个宽大的浅紫色总结框，标题为：
**如何理解这个界？**

总结框分成三列：

### 第一列
**结构项**
```
√(k·log M/n)
```
**解释：** 由稀疏拓扑决定。k 越小，激活结构越少，组合复杂度越低。

### 第二列
**参数项**
```
√(d_eff·log(C_eff·L_eff·n)/n)
```
**解释：** 由有效参数维度决定。物理约束使参数搜索不再发生在无约束高维空间。

### 第三列
**样本规模**
```
n ↑ ⇒ R̂_S(H) ↓
```
**解释：** 样本越多，复杂度项被样本平均效应进一步压低。

---

## 右侧补充说明：逻辑约束的位置

右侧可以加一个窄小虚线注释框，标题为：
**逻辑相容性的作用**

内容写：
- 训练期：作为软约束改变优化解
- 推理期：作为规则满足度审查诊断边界
- 若写成硬可行集，可进一步收缩覆盖数

**注意：** 不要把逻辑约束强行画成最终复杂度界中的第三个显式加法项。它在本文中主要通过约束优化解和规则满足度影响可行解空间。

## 底部结论条

底部使用深紫色横向总结条，白色加粗文字：

**结论：NSNet 的复杂度压缩来自两条主线——稀疏结构减少可选路径，物理参数降低有效自由度；二者共同降低经验复杂度项，为泛化倾向提供统计学习解释。**
